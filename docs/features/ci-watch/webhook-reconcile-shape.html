CI Watch Webhook Reconcile Shape

Webhooks are edge triggers. The backend does one authoritative reconcile for the watched PR head and transitions only on aggregate evidence. Polling is constrained to two explicit places: a bounded mergeability retry where GitHub reports UNKNOWN, and the current publish observer/backstop. The webhook handler itself does not spin.

No webhook spin loop GitHub's UNKNOWN or REST mergeable=null means calculation is not ready. The handler refreshes the watch detail and waits for another event or a deferred backstop instead of spinning.
Current polling surface publish_current_head also starts a background commit observer that samples CI and mergeability about every 30 seconds, currently up to 60 attempts, until failure, green+mergeable, or timeout. That is a backstop/ledger observer, not the webhook wake path.
