Aggregate Reconcile

This is the event-triggered control loop. A webhook is only an edge notification; the backend recomputes aggregate CI and PR state before changing the durable watch.

Overview Agent handoff Webhook trigger Aggregate reconcile Terminal action
Read aggregate CI evidence Latest check run per name, combined status, and path-filtered prior-green evidence.
Read PR mergeability Use GitHub's PR state to detect dirty or behind. UNKNOWN remains pending; draft is not a conflict.
Classify the aggregate failed checks -> red; dirty or behind -> conflict; CI satisfied plus clean -> ready; missing checks or UNKNOWN mergeability -> pending.
No webhook spin loop GitHub's UNKNOWN or REST mergeable=null means "calculation not ready". The handler refreshes the watch detail and waits for another event or a deferred backstop instead of spinning.
Current polling surface publish_current_head also starts a background commit observer that samples CI and mergeability about every 30 seconds, currently up to 60 attempts, until failure, green+mergeable, or timeout. That is a backstop/ledger observer, not the webhook wake path.
